‚óŹ Great! Let me break down each component of that DNS record and explain

  DNS Record Breakdown

  Type: A    Name: @    IPv4: 185.199.108.153    Proxy: Off (grey cloud)

  ---
  Type: A

  What it means: "Address" record - maps a domain name to an IPv4 address

  How it works:
  - When someone types skyenet.info in their browser
  - DNS system looks up: "What IP address does skyenet.info point to?"
  - The A record answers: "185.199.108.153"
  - Browser then connects to that IP address

  Analogy: Like a phone book entry - "Skyenet Corp" ‚Üí "555-0108"

  Other common types:
  - AAAA - Maps domain to IPv6 address (like A, but for newer IP format)
  - CNAME - Maps domain to another domain name (alias)
  - MX - Mail server records
  - TXT - Text records (used for verification, SPF, etc.)
  - NS - Name server records (explained below)

  ---
  Name: @

  What it means: The root/apex of your domain (the bare domain itself)

  In your case: @ represents skyenet.info (not www.skyenet.info or
  mail.skyenet.info, just the base domain)

  Other examples:
  - Name: @ ‚Üí Points skyenet.info to 185.199.108.153
  - Name: www ‚Üí Would point www.skyenet.info to something
  - Name: api ‚Üí Would point api.skyenet.info to something
  - Name: blog ‚Üí Would point blog.skyenet.info to something

  Why the @ symbol?
  - It's DNS notation for "the current origin"
  - In BIND (standard DNS server software), @ is shorthand for the domain
  itself
  - Cloudflare uses this convention

  Full DNS terminology:
  - @ = apex/root/naked domain
  - Anything else = subdomain

  ---
  IPv4: 185.199.108.153

  What it means: The IP address that the domain should resolve to

  This specific IP: Owned by GitHub and used for GitHub Pages hosting

  Why 4 different A records?
  185.199.108.153
  185.199.109.153
  185.199.110.153
  185.199.111.153

  Multiple records provide:
  1. Redundancy - If one server is down, others still work
  2. Load balancing - DNS can rotate through IPs to distribute traffic
  3. Geographic routing - Some DNS servers return the closest IP

  How multiple A records work:
  - Your DNS query might return all 4 IPs
  - Or it might return them in rotating order (round-robin DNS)
  - Your browser picks one and tries to connect
  - If it fails, it tries another
